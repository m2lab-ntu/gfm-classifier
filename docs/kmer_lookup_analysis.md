# k-mer Lookup Analysis — 為什麼 MT 13-mer 達 98.7%?(論文可用文字 + 數據)

> **目的**:回應「MT 13-mer 的高分是否只是 k-mer 查表(近似 Kraken)」。
> **給論文那個 chat**:下方「論文段落草稿」可直接插入 tokenization 討論/limitations。
> 這個 chat 不動論文 repo(避免雙 push 衝突)。

## 實驗設定
- 測試集:`clean_common`(99,742 條 150bp 讀序,與訓練 50M∪250M 依 seq-id 不相交)。
- 參考庫:5,000 萬條訓練讀序(reads_50M),與 MT-50M 同資料密度。
- 三個**零神經網路** baseline,同測試集/參考庫:
  1. **唯一鍵查表(Kraken 式)**:只用「屬唯一」13-mer 多數決。
  2. **組成眾數投票**:每個 13-mer 指到其最常見屬,讀序取多數決。
  3. **多項式 k-mer Naive Bayes**:用完整 P(13-mer|屬) 分布(α=1 平滑)。

## 結果(genus Top-1)
| 方法 | 準確率 |
|---|---|
| 唯一鍵查表(Kraken 式) | **0.72%** |
| 組成眾數投票 | **35.3%** |
| 多項式 k-mer Naive Bayes(α=1) | **74.9%** |
| **MT 13-mer(神經網路,50M)** | **87.5%** |
| MT 13-mer(神經網路,250M) | 98.7% |
| NT-v2 6-mer(498M 預訓練) | 67% |

## 關鍵觀察
- **k=13 沒有唯一鍵**:4¹³ = 6.71×10⁷ 的 13-mer 空間幾乎被 1,535 個基因體塞滿(參考中見到 6.70×10⁷ 個相異 13-mer)。測試讀序的 13-mer **100% 都在參考庫**,但只有 **0.4%** 為單一屬獨有。→ 與 Kraken(k=31,空間 4.6×10¹⁸ ≫ 基因體總量,故 31-mer 近乎唯一)本質不同。
- **查表全數失敗**:唯一鍵 0.72%(僅 1.36% 讀序有任何唯一 13-mer)、眾數投票 35.3%。→ MT 的高分**不是** k-mer 查表。
- **組成訊號極強**:正規多項式 NB 用完整 P(13-mer|屬) 分布達 **74.9%** → 13-mer 組成本身就高度可分屬(非查表,而是頻率剖面)。
- **神經網路仍加值 ~13pp**:MT 87.5% vs NB 74.9%(同 50M 參考)。NB 假設 token 獨立;MT 以 embedding + attention 捕捉**共現/依賴**,榨出 NB 抓不到的訊號。
- **★ tokenization > 模型**:零神經網路的 13-mer NB(74.9%)**贏過** 498M 參數、預訓練的 NT-v2 6-mer(67%)。表徵(k 長度)比模型容量/預訓練更決定性——直接支持主結論。

## 論文段落草稿(繁中,可直接插入)
> 為釐清 MetaTransformer 於重疊 13-mer tokenization 下達到 98.7\% 屬層級準確度的機制,我們檢驗其是否僅為 $k$-mer 精確比對(近似查表)。以 clean\_common 測試集之 99{,}742 條讀序為對象,並以 5{,}000 萬條訓練讀序建立 13-mer$\to$屬之參考對應。分析顯示,$4^{13}=6.71\times10^{7}$ 之 13-mer 空間幾乎被 1{,}535 個基因體完全覆蓋(參考中觀察到 $6.70\times10^{7}$ 個相異 13-mer);測試讀序之 13-mer 有 100\% 出現於參考,然而僅 0.4\% 為單一屬所獨有。因此在 $k=13$ 尺度下,並不存在如 Kraken($k=31$)所依賴之屬特異性唯一鍵。僅使用屬唯一 13-mer 之多數決分類器準確度為 0.72\%,將每個 13-mer 指派至其最頻繁屬別後之組成式多數決分類器為 35.3\%。然而,採用完整 $P(\text{13-mer}\mid\text{屬})$ 分布之多項式 Naive Bayes 分類器達 74.9\%,顯示 13-mer 之組成頻率剖面本身即高度屬可分;MetaTransformer 於相同 50M 參考下之 87.5\% 進一步高出約 13 個百分點,反映其以 token embedding 與注意力池化捕捉了 Naive Bayes 獨立性假設所忽略之 $k$-mer 共現結構。值得注意的是,此零神經網路之 13-mer Naive Bayes(74.9\%)已超越具 4.98 億參數且經預訓練之 NT-v2 於 6-mer tokenization 下之 67\%,顯示 tokenization 之選擇較模型容量與預訓練更具決定性。綜言之,MetaTransformer 之高準確度並非源於 $k$-mer 精確查表,而是對屬特異性組成與共現結構之表徵學習;tokenization 的關鍵作用在於決定**可學習表徵的維度與可分性**:6-mer 僅 4{,}096 詞彙,組成空間維度過低、各屬高度重疊,致準確度上限約 67\%;13-mer 之高維詞彙則提供屬可分之組成空間。惟須強調,此分析僅證明模型並非執行平凡查表,並未證明其可泛化至參考庫外之新基因體;13-mer 空間於閉集下之飽和,反而凸顯對未見基因體之泛化能力仍待專門評估。
